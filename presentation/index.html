01

Echo —
минималистичная соцсеть-мессенджер

Терминальный стиль. Два формата в одном приложении: короткие посты в ленте и приватные чаты один-на-один / группы. Фокус на тексте, без лишнего шума.

Frontend

React 19 + TS

Vite, React Router, TanStack Query, Zustand, Zod, Axios, MSW.

Backend

Django 6 + DRF

SimpleJWT, Channels (WebSocket), SQLite для dev.

Качество

Тесты

Vitest (unit/integration), Playwright E2E на 3 viewport'ах.

Что говорить: «Коротко: соцсеть в стиле терминала — лента плюс чаты, full-stack на React+Django, с реальным WebSocket и тестами».
02

Архитектура: слоистая и предсказуемая

Frontend разделён по слоям (feature-sliced): запросы не уходят в обход API-клиента, валидация — на границе. Backend — REST + WebSocket поверх единой JWT-сессии.

FRONTEND
app pages widgets features shared (api/zod/ui)
▼ HTTPS / WSS
BACKEND
users chats (DRF + Channels) feed ai core (errors)
ХРАНИЛИЩЕ / АУТЕНТИФИКАЦИЯ
SQLite JWT (1d access / 7d refresh) HttpOnly refresh cookie
Что говорить: «Слоистая архитектура: app → pages → widgets → features → shared. Бизнес-логика в сериализаторах, не во вьюсетах. Ошибки приводятся к единому формату {code,message}».
03

Ключевые фичи

Что уже работает end-to-end и закрыто тестами.

🔐 JWT-аутентификация

ProtectedRoute / PublicOnlyRoute, refresh-токен в HttpOnly-куке, persist в localStorage.

💬 Чаты 1-на-1 и группы

WebSocket через Django Channels, история сообщений, индикатор непрочитанных.

📡 Лента с cursor-пагинацией

Посты, лайки, превью. Курсорная пагинация в TanStack Query.

📱 Mobile-first layout

<720px — TabBar, ≥720px — Navbar. Один и тот же UX на телефоне и десктопе.

🛡 Zod на границе

Все исходящие payload и входящие ответы валидируются — drift бэкенда ловится сразу.

🧪 MSW-моки

UI можно разрабатывать без поднятого бэкенда — фронт живёт на MSW-обработчиках.

Что говорить: «Не MVP-фичи ради галочки — каждая закрыта E2E-тестом и реальным контрактом API».
04

Контракт API — единый и явный

Все маршруты — без trailing slash. Ошибки приводятся к плоскому JSON. Это сознательное решение, зафиксированное и в роутере Django, и в axios-клиенте. 

# формат ошибки (одинаковый везде)
{
  "code":    "invalid_credentials",
  "message": "wrong email or password",
  "detail":  "..."
}

# ключевые маршруты
POST   /api/auth/register
POST   /api/auth/login            → JWT
GET    /api/users/me
GET    /api/chats                 # список
POST   /api/chats
GET    /api/chats/{id}/messages
POST   /api/chats/{id}/messages
POST   /api/chats/{id}/read
GET    /api/feed?cursor=...
WSS    /ws/chat/{id}/             # realtime
Что говорить: «URL без слэша в конце — это часть контракта. Если фронт или тест добавит /, получит 301 — и это сигнал о нарушении контракта».
